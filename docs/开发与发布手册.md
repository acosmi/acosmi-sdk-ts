# acosmi-sdk-ts 开发与发布手册

> 适用版本: v0.19.0+
> 最后更新: 2026-05-01

## 1. 项目概览

`acosmi-sdk-ts` 是 Acosmi 模型网关的 TypeScript SDK,从 `acosmi-sdk-go` 全量端口而来。

| 项目  | 值 |
| ----- | -- |
| 端口源 | `acosmi-sdk-go` v0.19.0 |
| 端口策略 | bug-for-bug 一字不差对齐 |
| 版本号 | 严格对齐 Go SDK |
| 主消费者 | `crabcode` (Anthropic 格式) |
| 长期保留 | `OpenAIAdapter` (`crabclaw` 暂不用 TS 但代码须与 Go 对齐) |

## 2. 仓库与发布拓扑

```
源码权威源 (private monorepo)         公开镜像仓 (public)              npm 公开包
D:\chat-acosmi                  →    github.com/acosmi/acosmi-sdk-ts  →   npmjs.com/package/acosmi-sdk-ts
└── acosmi-sdk-ts/                   (subtree push 同步, 含源码)        (npm publish, 仅 dist)
                                     └── 不接受 PR (镜像)
                                                      ↓
                                              下游 crabcode
                                              npm install acosmi-sdk-ts
```

**铁律**: 改源码**只在主仓**改。公开仓由 `git subtree push` 同步,不直接修改。

## 5. 编码规范

### TypeScript 风格 (硬规则)

| 类别 | 规则 | 例 |
|------|------|----|
| 函数/方法 | **camelCase** | `discover`, `getAdapterForModel`, `applyRequestSanitizers` |
| 类 / 接口 / 类型 | **PascalCase** | `Client`, `ServerMetadata`, `RetryPolicy`, `AnthropicAdapter` |
| Error 子类 | **PascalCase + Error 后缀** | `HTTPError`, `NetworkError`, `BusinessError` |
| 常量 | **PascalCase** (前缀型) 或 **UPPER_CASE** | `ErrDiscovery`, `ScopeAI`, `EventComplete` |
| 文件名 | **kebab-case** | `stream-meta.ts`, `bug-report.ts` |
| Wire-format 字段 | **snake_case** (与 Go json tag 严格对齐) | `preferred_format`, `created_at`, `max_tokens` |

### 不要使用的 Go-isms (硬禁止)

- ❌ **PascalCase 函数** — TS 公开 API 必须 camelCase (`AllScopes()` 是 Go 风格,改 `allScopes()`)
- ❌ **多返回值 `[T, Error]` 元组** — 用 `throw` + 类型化 Error 子类
- ❌ **`nil` 字面量** — 用 `null` / `undefined`
- ❌ **`: any`** — 优先 `: unknown` 或精确类型,确实需要再 `any`
- ❌ **Go 关键字识别符**: `func`, `package`, `chan`, `defer`, `goroutine`
- ❌ **指针风格 `T | null`** — 用 optional `?: T` (除非 wire-format 必须)
- ❌ **`fmt.Sprintf` / `fmt.Errorf` / `log.Println`** — 用模板字符串 / `new Error()` / `console`

### 保留的 Go 印记 (有意为之, 不许改)

- ✅ **snake_case wire-format 字段名** — 与 Go json tag 一字不差
- ✅ **"端口自 X.go" / "// Port from..." 溯源注释** — 长期维护对照用
- ✅ **ISO 8601 字符串时间** — 与 Go RFC3339 wire-compat
- ✅ **string 表示金额 / json.Number 字段** — 避免 JS number 精度损失
- ✅ **bug-for-bug 行为对齐** (e.g. retry POST=false 默认, sanitize thinking 硬豁免)
- ✅ **双 adapter 等地位** — `AnthropicAdapter` + `OpenAIAdapter` P0 红线,不可降级

### TS 不可避免的偏移 (与 Go 不 1:1)

| 维度 | Go | TS |
|------|----|----|
| 错误处理 | `(val, err)` 多返回值 | `throw` + 类型化 Error 子类 |
| 上下文取消 | `context.Context` | `AbortSignal` |
| 流返回 | `(eventCh, errCh chan)` | `AsyncIterable<StreamEvent>` |
| 锁 | `sync.Mutex` | Promise chain (`this.mu`) |
| IO | 同步 `os.ReadFile` | `fs/promises` async |

## 6. 开发流程

### 改源码 → 提交

```bash
# 主仓 D:\chat-acosmi
cd acosmi-sdk-ts/

# 改完后必须四绿才提交
npm run typecheck    # 0 errors
npm run lint         # 0 errors
npm test             # 36/36 pass
npm run build        # 多端 dist/ 全部生成

# commit (主仓根目录)
cd ..
git add acosmi-sdk-ts/
git commit -m "fix(sdk-ts): ..."
```

### Commit message 规范

`type(scope): subject` 风格 (与 Go SDK 仓库一致):

- `feat(sdk-ts): ...` 新功能
- `fix(sdk-ts): ...` bug fix
- `refactor(sdk-ts): ...` 重构 (无功能变化)
- `docs(sdk-ts): ...` 文档
- `test(sdk-ts): ...` 测试
- `release: vX.Y.Z` 版本发布

## 7. 测试

- 框架: **vitest** (Node + jsdom 兼容)
- 位置: `test/`
- 命令: `npm test` 单跑 / `npm run test:watch` watch

- `test/adapters/routing.test.ts` — 双格式四级路由 8 case (P0 红线)
- `test/adapters/anthropic-build.test.ts` — buildRequestBody 14 case
- `test/sanitize/history.test.ts` — StripEphemeral / DropBlocks / thinking 豁免 7 case

加测试规则:
- 端口自 Go SDK 的测试用例必须保留 bug-for-bug
- TS-only 测试 (e.g. AbortSignal 行为) 单独写文件,不混进端口测试

## 9. 发布流程

### 一次性配置 (首次发布前)

1. **创建 GitHub repo**: `github.com/acosmi/acosmi-sdk-ts` (public)
2. **配置主仓 subtree remote**:
   ```bash
   cd D:\chat-acosmi
   git remote add ts-public git@github.com:acosmi/acosmi-sdk-ts.git
   ```
3. **npm.js automation token**:
   - 登录 npmjs.com,创建 automation token
   - GitHub repo → Settings → Secrets → 新增 `NPM_TOKEN`
4. **首次 subtree push**:
   ```bash
   git subtree push --prefix=acosmi-sdk-ts ts-public main
   ```

### 每次发布

```bash
# 1. 主仓改完源码,四绿
cd D:\chat-acosmi\acosmi-sdk-ts
npm run typecheck && npm run lint && npm test && npm run build

# 2. 升 version (与 Go SDK 严格对齐)
# 编辑 package.json "version" 字段, e.g. 0.19.0 → 0.19.1

# 3. commit + subtree push
cd D:\chat-acosmi
git add acosmi-sdk-ts/
git commit -m "release(sdk-ts): v0.19.1"
git subtree push --prefix=acosmi-sdk-ts ts-public main

# 4. 在公开仓打 tag (触发 CI)
git clone git@github.com:acosmi/acosmi-sdk-ts.git /tmp/sdk-ts-release
cd /tmp/sdk-ts-release
git tag v0.19.1
git push origin v0.19.1

# 5. GitHub Actions 自动:
#    - npm ci → typecheck → lint → test → build
#    - npm publish --provenance --access public
#    - 创建 GitHub Release (auto release notes)

# 6. 验证
npm view acosmi-sdk-ts version    # 应显示 0.19.1
```

## 10. 版本号策略

- **严格对齐 Go SDK 版本号**
- Go v0.19.0 → TS v0.19.0
- Go v0.19.1 (patch) → TS v0.19.1
- Go v1.0.0 (major) → TS v1.0.0 (做 diff 同步, 重大迭代)
- TS-only hotfix: 用 build metadata `0.19.0+ts.1` (待商榷)

## 11. Go SDK 同步流程

每次 Go SDK 升级:

1. 看 Go release notes 改动列表
2. `git diff acosmi-sdk-go vX.Y.Z..vX.Y.Z+1` — 看实际代码改动
3. 镜像改动到 acosmi-sdk-ts (保留 TS 不可避免偏移, 见 §5)
4. 如改动涉及 wire-format 字段, 同步更新 `src/types.ts`
5. 跑四绿 (typecheck/lint/test/build)
6. 升 TS version 对齐
7. 走发布流程 §9

## 12. 故障排查

### `npm publish` 失败

| 错误 | 排查 |
|------|------|
| `403 Forbidden` | `NPM_TOKEN` secret 失效 / 无权限 |
| `package already exists` | 没升 version |
| `private package can't publish` | `package.json` 还有 `"private": true` |
| provenance 失败 | 临时去掉 workflow 里 `--provenance` flag |

### typecheck 失败

- 改了 `src/types.ts` wire-format → 检查所有 caller
- import 路径错 → 用 `npm run typecheck` 定位
- TS strict 默认开,看具体 error message

### test 失败

- `routing.test.ts` 红 = **P0 红线破坏, 严重停车**
- `history.test.ts` 红 = sanitize 行为漂移, 检查 thinking 豁免 / tool_use_id 联动
- `anthropic-build.test.ts` 红 = adapter buildRequestBody 行为变了

### subtree push 冲突

```bash
# 公开仓不接受直接修改, 如有人误改:
# 1. 主仓必须是唯一权威源
# 2. force push (谨慎确认无合法改动):
git subtree push --prefix=acosmi-sdk-ts ts-public main --force
```

### 下游 crabcode 报 type 错

- 检查 `package.json` exports 字段 `types` 字段是否齐
- `dist/node/index.d.ts` 必须存在 (build 产物)
- 用 `npm pack --dry-run` 看实际 publish 内容
