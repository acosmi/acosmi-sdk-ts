| 长期保留 | `OpenAIAdapter` (P0 红线，`crabclaw` 暂不用 TS 但代码与 Go 对齐) |
| LICENSE  | MIT — Copyright (c) 2026 Acosmi |

## 5. 编码规范

### TypeScript 风格（硬规则）

| 类别 | 规则 | 例 |
|------|------|----|
| 函数/方法 | **camelCase** | `discover`, `getAdapterForModel` |
| 类 / 接口 / 类型 | **PascalCase** | `Client`, `ServerMetadata`, `AnthropicAdapter` |
| Error 子类 | **PascalCase + Error 后缀** | `HTTPError`, `BusinessError`, `OrderTerminalError` |
| 常量 | **PascalCase**（前缀型）或 **UPPER_CASE** | `ErrDiscovery`, `ScopeAI`, `EventComplete` |
| 文件名 | **kebab-case** | `stream-meta.ts`, `bug-report.ts` |
| Wire-format 字段 | **snake_case**（与 Go json tag 严格对齐） | `preferred_format`, `created_at`, `max_tokens` |

### 不要使用的 Go-isms（硬禁止）

- ❌ **PascalCase 函数** — TS 公开 API 必须 camelCase（`AllScopes()` 是 Go 风格，改 `allScopes()`）
- ❌ **多返回值 `[T, Error]` 元组** — 用 `throw` + 类型化 Error 子类
- ❌ **`nil` 字面量** — 用 `null` / `undefined`
- ❌ **`: any`** — 优先 `: unknown` 或精确类型
- ❌ **Go 关键字识别符**：`func`, `package`, `chan`, `defer`, `goroutine`
- ❌ **指针风格 `T | null`** — 用 optional `?: T`（除非 wire-format 必须）
- ❌ **`fmt.Sprintf` / `fmt.Errorf` / `log.Println`** — 用模板字符串 / `new Error()` / `console`

### 保留的 Go 印记（有意为之，不许改）

- ✅ **snake_case wire-format 字段名** — 与 Go json tag 一字不差（50+ struct 已对齐）
- ✅ **"端口自 X.go" / "// Port from..." 溯源注释** — 长期维护对照用
- ✅ **ISO 8601 字符串时间** — 与 Go RFC3339 wire-compat
- ✅ **string 表示金额 / json.Number 字段** — 避免 JS number 精度损失（金融安全）
- ✅ **bug-for-bug 行为对齐**（e.g. retry POST=false 默认，sanitize thinking 硬豁免）
- ✅ **双 adapter 等地位** — `AnthropicAdapter` + `OpenAIAdapter` P0 红线，不可降级

### TS 不可避免的偏移（与 Go 不 1:1）

| 维度 | Go | TS |
|------|----|----|
| 错误处理 | `(val, err)` 多返回值 | `throw` + 类型化 Error 子类 |
| 上下文取消 | `context.Context` | `AbortSignal` |
| 流返回 | `(eventCh, errCh chan)` | `AsyncIterable<StreamEvent>` |
| 锁 | `sync.Mutex` | Promise chain (`this.mu`) |
| IO | 同步 `os.ReadFile` | `fs/promises` async |
| Token Store 持久化 | 同步 file IO | `async` Save/Load/Clear |

4. Permissions：Read & Write
5. Packages：选 `@acosmi/sdk-ts`
6. **勾选 "Bypass two-factor authentication when publishing"** ← 漏勾就 E403
7. 复制 token

**GitHub repo Secret**：
- repo → Settings → Secrets and variables → Actions → New repository secret
- Name: `NPM_TOKEN`
- Value: 上一步 token

## 10. 版本号策略

**自 v1.0.0 起 Go SDK + TS SDK 联动稳定测试版基线，主版本号同步，patch 号独立**。

- **联动 minor / major**（1.x.0 / x.0.0）：Go + TS 同步发布稳定基线（如 1.0.0 / 1.1.0 / 2.0.0）
- **独立 patch**（1.x.y）：TS 独立 patch fix（bug fix、文档、内部 refactor）— 不必对齐 Go SDK 的 patch 号
- major（x.0.0）：wire-format 不兼容变更、API 移除、TokenStore 接口变化 — Go + TS 必须同步升

Go SDK 升级时：参考改动列表（见 §11），主版本必须同步，patch 按需独立。

## 11. Go SDK 同步流程

每次 Go SDK 升级（主版本联动，patch 独立）：

1. 看 Go SDK release notes / `git diff` 改动列表
2. 评估对 TS 影响：
   - **必同步**：wire-format 字段（`src/types.ts`）、新业务方法（`client/*.ts`）、双格式路由逻辑
   - **可选**：内部 refactor、Go-only 优化（goroutine pool 等）、CLI 工具（`cmd/` 下不端口）
3. 镜像改动到 `acosmi-sdk-ts/`，保留 TS 不可避免偏移（见 §5）
4. 跑四绿（typecheck/lint/test/build）
5. 升 TS version：Go SDK 主版本变（1.0→1.1）→ TS 必须同步升 minor；Go SDK 仅 patch 变 → TS 按需独立 patch
6. 走发布流程 §9.2

## 12. 故障排查

### 12.1 npm publish 失败

| 错误 | 原因 | 排查 |
|------|------|------|
| `403 Forbidden` "Two-factor authentication or granular access token with bypass 2fa enabled is required" | Token 没开 bypass 2FA | 重建 Granular token，**勾选 "Bypass two-factor authentication"**；或换 Classic Automation token |
| `403 Forbidden`（其他） | `NPM_TOKEN` secret 失效 / 包名权限不够 | 重建 token，确保 packages 选了 `@acosmi/sdk-ts` |
| `EPUBLISHCONFLICT` / `cannot publish over the previously published versions` | 没升 version | 升 `package.json` `"version"` |
| `package private` | `package.json` 还有 `"private": true` | 删掉或改 `false` |
| `provenance attestation failed` | 仓库 visibility / OIDC 配置不对 | workflow 里**去掉** `--provenance` flag |
| `404 Not Found` 在 publish 时 | 包名拼错或 `publishConfig.access` 缺 | `package.json` 加 `"publishConfig": { "access": "public" }` |

### 12.2 typecheck 失败

- 改了 `src/types.ts` wire-format → 检查所有 caller
- import 路径错 → `npm run typecheck` 输出文件:行号
- TS strict 默认开 → 看具体 error，不要图方便加 `as any`

### 12.3 test 失败

- `routing.test.ts` 红 = **P0 红线破坏，严重停车**，立刻回滚
- `history.test.ts` 红 = sanitize 行为漂移（检查 thinking 豁免 / tool_use_id 联动）
- `anthropic-build.test.ts` 红 = adapter buildRequestBody 行为变了（核对 Go SDK 是否同步漂移）

### 12.4 公开仓 push 失败

| 错误 | 排查 |
|------|------|
| `Host key verification failed` | 把 remote URL 从 `git@github.com:` 改成 `https://github.com/` |
| `Updates were rejected because the remote contains work` | 用 `git push --force`（公开仓单 commit 模型，force 是常规操作） |
| `This repository moved. Please use the new location` | 仓库被 transfer 了，更新 remote URL（见原始 enterprise repo） |
| GitHub Contributors 还显示 Claude | 缓存延迟，等 5-10 分钟后看；或 force push 后强制刷新 |

### 12.5 下游 crabcode 报 type / 模块 缺失

- 检查 `package.json` `exports` 字段 `types` 是否齐
- `dist/node/index.d.ts` 必须存在（build 产物）
- `npm pack --dry-run` 看实际 publish 内容
- 用户问"缺 X" → 先核 Go SDK 是否真有 → 再核 `D:\chat-acosmi\acosmi-sdk-ts\src` 是否端口 → 多数是 README 未文档化（更新 README）
